모든 API 엔드포인트는 API 키를 사용한 인증이 필요합니다.

## API 키 발급

1. [BIZ MORI 개발자 센터](https://developers.mori.art)에 로그인합니다
2. **설정** > **API 키**로 이동합니다
3. **API 키 생성**을 클릭합니다
4. 키를 복사하여 안전하게 보관합니다 — 한 번만 표시됩니다

<Warning>
  API 키는 계정의 API에 대한 전체 접근 권한을 부여합니다. 클라이언트 측 코드나 공개 저장소에 절대 노출하지 마세요.
</Warning>

## API 키 사용 방법

`Authorization` 헤더에 Bearer 형식으로 API 키를 포함합니다:

<CodeGroup>
```bash cURL
curl -X POST https://morimori.app/api/v2/orders/anti-ai \
  -H "Authorization: Bearer YOUR_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"idempotencyKey": "unique-key", "files": [{"fileName": "image.jpg"}]}'
```

```javascript Node.js
const response = await fetch('https://morimori.app/api/v2/orders/anti-ai', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    idempotencyKey: 'unique-key',
    files: [{ fileName: 'image.jpg' }],
  }),
});
```

```python Python
import requests

response = requests.post(
    'https://morimori.app/api/v2/orders/anti-ai',
    headers={
        'Authorization': 'Bearer YOUR_API_TOKEN',
        'Content-Type': 'application/json',
    },
    json={
        'idempotencyKey': 'unique-key',
        'files': [{'fileName': 'image.jpg'}],
    },
)
```
</CodeGroup>

## 요청 제한

API 요청은 구독 플랜에 따라 제한됩니다:

| 플랜 | 분당 요청 수 |
|------|------------|
| 무료 | 60 |
| Pro | 300 |
| Enterprise | 1,000 |

요청 제한을 초과하면 API는 `RATE_LIMIT_EXCEEDED` 오류 코드와 함께 `429` 상태 코드를 반환합니다.

## 멱등성 (Idempotency)

모든 주문 생성 엔드포인트는 `idempotencyKey` 파라미터를 지원합니다. 동일한 키로 중복 요청을 보내면 새 주문을 생성하는 대신 기존 주문을 반환합니다. 이를 통해 네트워크 재시도 시 이중 처리를 방지할 수 있습니다.

```json
{
  "idempotencyKey": "unique-request-id-123",
  "files": [...]
}
```
