이 가이드는 가장 일반적인 사용 사례인 Anti-AI 보호 주문 생성을 안내합니다. 워터마크 삽입 주문에도 동일한 흐름이 적용됩니다.

## 사전 준비

- MORI API 키 ([여기서 발급](https://developers.mori.art))
- 보호할 이미지 파일 (jpg, jpeg, png, 또는 webp)

## 1단계: 주문 생성

원하는 입력 방식을 선택하세요:

<Tabs>
  <Tab title="업로드 모드">
    주문을 생성하고 S3에 파일을 업로드하기 위한 Presigned URL을 받습니다.

    ```bash
    curl -X POST https://morimori.app/api/v2/orders/anti-ai \
      -H "X-API-Key: sk_your_api_key" \
      -H "Content-Type: application/json" \
      -d '{
        "idempotencyKey": "my-first-order-001",
        "files": [
          { "fileName": "photo.jpg" }
        ],
        "options": {
          "strength": "high"
        }
      }'
    ```

    **응답:**
    ```json
    {
      "data": {
        "orderName": "anti_ai_2026-02-19",
        "orderId": "123456789",
        "status": "pending",
        "files": [
          {
            "fileId": 1,
            "fileName": "photo.jpg",
            "uploadUrl": "https://s3.amazonaws.com/...",
            "fileKey": "temp/123456789/0/photo.jpg"
          }
        ]
      }
    }
    ```
  </Tab>
  <Tab title="URL 모드">
    이미지 URL을 직접 제공합니다 — 파일 업로드 없이 즉시 처리가 시작됩니다.

    ```bash
    curl -X POST https://morimori.app/api/v2/orders/anti-ai \
      -H "X-API-Key: sk_your_api_key" \
      -H "Content-Type: application/json" \
      -d '{
        "idempotencyKey": "my-first-order-001",
        "files": [
          {
            "fileName": "photo.jpg",
            "originalFileUrl": "https://your-server.com/photo.jpg"
          }
        ],
        "options": {
          "strength": "high"
        }
      }'
    ```

    **응답:**
    ```json
    {
      "data": {
        "orderName": "anti_ai_2026-02-19",
        "orderId": "123456789",
        "status": "inProgress",
        "files": [
          {
            "fileId": 1,
            "fileName": "photo.jpg",
            "originalFileUrl": "https://your-server.com/photo.jpg"
          }
        ]
      }
    }
    ```

    <Info>
      URL 모드를 사용하면 [4단계](#4단계-주문-상태-확인)로 바로 이동하세요 — 업로드 및 확인 단계가 필요 없습니다.
    </Info>
  </Tab>
</Tabs>

## 2단계: 파일 업로드 (업로드 모드만 해당)

응답에서 받은 Presigned URL로 이미지를 업로드합니다:

```bash
curl -X PUT "https://s3.amazonaws.com/..." \
  -H "Content-Type: image/jpeg" \
  --data-binary @photo.jpg
```

<Note>
  Presigned URL은 일정 시간 후 만료됩니다. 만료된 경우 [URL 갱신](/ko/api-reference/orders/refresh-urls) 엔드포인트를 사용하여 새 URL을 받으세요.
</Note>

## 3단계: 주문 확인 (업로드 모드만 해당)

모든 파일 업로드 완료 후, 처리를 시작하기 위해 주문을 확인합니다:

```bash
curl -X POST https://morimori.app/api/v2/orders/anti-ai/confirm \
  -H "X-API-Key: sk_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "idempotencyKey": "my-confirm-001",
    "orderId": "123456789"
  }'
```

## 4단계: 주문 상태 확인

주문 상태를 폴링하거나 [웹훅](/ko/webhooks)을 설정하여 알림을 받습니다.

```bash
curl https://morimori.app/api/v2/orders/123456789 \
  -H "X-API-Key: sk_your_api_key"
```

주문은 다음 상태를 거칩니다:
- `pending` → 파일 업로드 대기 중
- `inProgress` → 처리 중
- `complete` → 완료 (다운로드 가능)
- `failed` → 처리 실패

## 5단계: 결과 다운로드

상태가 `complete`이면 다운로드 URL을 받습니다:

```bash
curl https://morimori.app/api/v2/orders/123456789/download \
  -H "X-API-Key: sk_your_api_key"
```

**응답:**
```json
{
  "data": {
    "downloadUrl": "https://s3.amazonaws.com/...",
    "expiresIn": 3600
  }
}
```

Presigned URL을 사용하여 보호된 파일을 다운로드합니다.
